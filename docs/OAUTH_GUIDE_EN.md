# Supabase OAuth Configuration Guide (GitHub & Google)

To allow users to sign in using their Google or GitHub accounts, you will need to configure the OAuth providers in your Supabase dashboard and obtain the necessary keys from each platform.

---

## 1. GitHub OAuth Configuration

### Step A: Create an OAuth App in GitHub

1. Go to [GitHub Developer Settings](https://github.com/settings/developers).
2. On the left menu, click on **OAuth Apps**.
3. Click the **New OAuth App** button.
4. Fill in the form:
   - **Application name**: The name of your app (e.g., `UrbanPulse`).
   - **Homepage URL**: The main URL of the site (e.g., `http://localhost:3000` for local development or your Vercel address for production).
   - **Authorization callback URL**: This URL is crucial! This is where GitHub will send the user after authentication. It must have the following format:
     - `https://<PROJECT_REF>.supabase.co/auth/v1/callback`
     - You can find your specific project URL (`PROJECT_REF`) in Supabase: go to **Project Settings** -> **API**, section **Project URL**.
5. Click **Register application**.
6. You will now see a **Client ID**. Copy it, you will need it shortly.
7. Click **Generate a new client secret** (you may have to re-enter your GitHub password).
8. Copy the generated **Client Secret**. Be careful, you won't be able to see it again after leaving this page.

### Step B: Configuration in Supabase

1. Go to the [Supabase Dashboard](https://supabase.com/dashboard).
2. Select your project (e.g., `urbanpulse-hackathon`).
3. On the left menu, go to **Authentication** -> **Providers**.
4. Look for **GitHub** in the list and click on it to enable it.
5. Toggle the **Enable GitHub** option.
6. Enter the **Client ID** you copied earlier from GitHub.
7. Enter the **Client Secret** generated by GitHub.
8. Click **Save**.

## 2. Google OAuth Configuration

### Step A: Create a Project and Obtain Keys from Google Cloud Console

1. Go to [Google Cloud Console](https://console.cloud.google.com/).
2. If you don't already have a project, create one by clicking the drop-down at the top left (next to the logo) and selecting **New Project**. Give it a name (e.g., `UrbanPulse`) and click **Create**.
3. Select the newly created project from the drop-down.
4. In the side menu (hamburger menu), navigate to **APIs & Services** -> **Credentials**.
5. Before creating a Client ID, you must configure the "OAuth consent screen" (the screen the user sees when asked for permission). Click the **Configure Consent Screen** button.
   - Choose **External** (if you want anyone to be able to authenticate, which is most likely) and click **Create**.
   - Fill in the required fields:
     - **App name**: e.g., `UrbanPulse`.
     - **User support email**: Your email.
     - **Developer contact information**: Your email.
   - Click **Save and Continue**.
   - For the next steps (Scopes, Test users) you can just click **Save and Continue** for now.
   - Go back to the Dashboard or directly to the **Credentials** menu.
6. Now click on the top button **+ CREATE CREDENTIALS** and choose **OAuth client ID**.
7. For **Application type**, select **Web application**.
8. Provide an identifying name (e.g., `Supabase Auth`).
9. Under **Authorized redirect URIs**, click **+ ADD URI**. Here you will need to re-enter the callback URL generated by Supabase, which has the format:
     - `https://<PROJECT_REF>.supabase.co/auth/v1/callback`
10. Click **Create**.
11. A window will appear with your **Client ID** and your **Client Secret**. Copy both of them!

### Step B: Configuration in Supabase

1. In the [Supabase Dashboard](https://supabase.com/dashboard), with your project open.
2. Navigate back to **Authentication** -> **Providers**.
3. Look for **Google** in the list and click to open it.
4. Toggle the **Enable Google** option.
5. Enter the **Client ID** obtained from the Google Cloud Console.
6. Enter the **Client Secret**.
7. To allow users to use the Google One-Tap button (optional but recommended for user experience), you can also check the **Skip nonce checks** option.
8. Click **Save**.

## 3. Usage in Frontend (UrbanPulse)

In the frontend, the routes in the `app/api/auth/oauth/route.ts` folder have been configured to simplify calls:

To authenticate with **GitHub**, create a button that redirects the user to:
`/api/auth/oauth?provider=github`

To authenticate with **Google**, create a button that redirects to:
`/api/auth/oauth?provider=google`

That's it! The user will be taken to the provider's page, log in, and then Supabase will redirect them back to the internal route (which you have at `/api/auth/callback`), successfully logging them into your UrbanPulse app. The profile will be automatically created in the database upon initial login due to the Supabase trigger configured on the `auth.users` table (`handle_new_user`).
