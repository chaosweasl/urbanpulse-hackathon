# UrbanPulse — AI Context & Rules

## Project Overview

UrbanPulse is a hyper-local community web application for a hackathon competition.
It connects neighbors within a ~500m radius to share resources, skills, and
emergency alerts. The goal is to transform passive neighbors into an active,
resilient support network through "Pulses" (Emergency, Skill, Item requests),
a Skill & Resource Library, private messaging, and an AI-powered lost pet system.

## Tech Stack

- **Framework:** Next.js 16 (App Router only, NO Pages Router)
- **Language:** TypeScript (backend API routes), TSX (frontend pages/components)
- **UI:** Tailwind CSS v4 + shadcn/ui v4 (Radix Maia style, Hugeicons)
- **Database & Auth:** Supabase (Auth, PostgreSQL + PostGIS, Realtime, Storage)
- **Maps:** Leaflet.js (interactive map with custom markers and heatmap)
- **Weather:** OpenWeatherMap API (severe weather auto-pinning)
- **Package Manager:** pnpm
- **Deployment:** Vercel

## Code Rules

### General

- App Router patterns ONLY — never use Pages Router
- Server Components for data fetching, Client Components only when needed
  (useState, useEffect, event handlers, browser APIs)
- Named exports for components, default exports for pages
- Use `@/` import alias for all project imports
- PascalCase for component files: `PulseCard.tsx`, not `pulse-card.tsx`
- Tailwind for ALL styling — no custom CSS files
- Use `cn()` from `@/lib/utils` for conditional class merging

### API Routes

- All responses follow: `{ success: boolean, data?: T, error?: string }`
- Use types from `@/types` — never use `any`
- Validate ALL inputs server-side (use Zod schemas from `@/lib/validators`)
- Never expose user UUIDs in URLs — use slugs for public-facing endpoints
- Use Supabase client directly — no ORM layer
- Keep API keys (OpenWeatherMap, etc.) in env vars, proxy through API routes

### Database

- All tables must have: `id` (UUID), `created_at` (timestamptz)
- UUID primary keys everywhere
- Use Supabase `rpc()` for PostGIS/geospatial queries
- Never write raw SQL unless via `rpc()` functions
- Use Row Level Security (RLS) policies for access control

### Components

- NEVER edit files inside `components/ui/` — they are shadcn-generated
- Create custom components in the appropriate feature subdirectory
- Keep components focused and reusable
- Use types from `@/types` for all props

### Auth & Security

- Password encryption via Supabase Auth (bcrypt)
- Secure session management via cookies (handled by `proxy.ts` middleware)
- Role-based access: `user`, `verified`, `admin`
- "Verified Neighbor" badge after 3+ successful interactions with positive feedback
- Admin routes must verify `admin` role before processing

## Competition Features Map

### 1. Neighborhood Dashboard (Real-Time Feed)

- **Dynamic Feed** (7 pts): Community alerts with Emergency/Skill/Item types + urgency levels
- **Live Updates** (8 pts): Supabase Realtime WebSocket — new pulses appear without refresh
- **Notification Engine** (8 pts): System alerts for emergencies, Hero Alerts for matched requests
- **Interactive Map** (10 pts): Leaflet.js map with pulse markers, density heatmap

### 2. Skill & Resource Library

- **Location Filtering** (10 pts): PostGIS `ST_DWithin` filtering within configurable radius
- **Direct Communication** (10 pts): Private messaging between neighbors

### 3. Smart Request Matching

- **Localization** (5 pts): Profile neighborhood/area settings + Skill Tags
- **Account Management** (5 pts): Full CRUD profiles, data deletion
- **Reliability Logic** (8 pts): Trust Scores based on feedback + successful interactions

### 4. Verification & Moderation

- **Secure Auth** (5 pts): Supabase Auth with encrypted passwords + sessions
- **Input Validation** (5 pts): Zod schemas, server-side validation
- **Auto-Validation** (6 pts): Posts auto-verified when 3+ neighbors confirm
- **Admin Dashboard** (6 pts): Review flagged content, merge duplicates, manage users
- **Data Privacy** (7 pts): Role-based access, RLS, private addresses/messages protected

### Bonus: AI Pet Guardian

- Photo analysis for species, color, markings
- Similarity matching: found → lost database with confidence scores
- Match interface with side-by-side results

### Bonus: Scalability

- Stateless architecture + caching
- Async processing for AI/notifications
- Graceful degradation if secondary services fail
